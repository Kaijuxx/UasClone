Alasan JUnit 5 dan Mockito di jadikan satu dalam pengujian ini adalah bertujuan untuk melakukan isolasi pengujian (isolated unit testing), di mana JUnit 5 berperan sebagai eksekutor untuk memvalidasi hasil akhir kode (assert), sementara Mockito bertugas menyediakan repository tiruan (mocking) di dalam memori. Dengan menyatukan kedua framework ini, pengujian logika bisnis pada NoteService dapat berjalan secara mandiri dengan efisiensi tinggi (dalam hitungan milidetik) karena seluruh data diolah langsung di RAM, sekaligus menghilangkan ketergantungan penuh terhadap status aktif maupun konfigurasi database MySQL fisik di lingkungan lokal.


KESIMPULAN
Seluruh fungsi logika utama pada NoteService berhasil diuji 
menggunakan Unit Testing dengan JUnit 5 dan Mockito.

Pengujian berhasil memastikan bahwa:
1. Service layer dapat mengolah data objek catatan dengan benar 
   sebelum dirender ke halaman web.
2. Logika pengurutan data catatan berdasarkan pin terbanyak dan ID 
   terbaru via findAllByOrderByIsPinnedDescIdDesc() berjalan dengan mulus.
3. Fitur interaktif web seperti klik Toggle Pin, Reminder, dan 
   Toggle To-Do memiliki validasi logika yang kuat.
4. Objek database berhasil di-mock secara sempurna sehingga proses 
   testing mandiri dan tidak bergantung pada kondisi database MySQL lokal.
